# QuickSight BI 功能指南

## 概述

Amazon QuickSight 是一个云原生商业智能服务,可让您创建交互式仪表板并执行即席分析。本指南涵盖了您可以用于销售数据的关键 BI 功能。

## 可用数据集

基于示例查询,您可以访问:
- **按地区销售** - 区域绩效指标
- **热门产品** - 产品收入和销售量
- **客户细分** - 按细分的客户行为
- **每日销售趋势** - 时间序列销售数据
- **类别绩效** - 产品类别盈利能力

## 关键 QuickSight BI 功能

### 1. 可视化类型

**条形图**
- 用于: 按地区销售、热门产品、类别绩效
- 显示: 跨类别的比较
- 示例: 比较北部、南部、东部、西部地区的收入

**折线图**
- 用于: 每日销售趋势
- 显示: 随时间的趋势
- 示例: 逐日跟踪收入增长

**饼图/环形图**
- 用于: 客户细分分布、类别组合
- 显示: 部分与整体的关系
- 示例: 按客户细分的收入百分比

**表格**
- 用于: 详细产品列表、客户详情
- 显示: 具有多个维度的粒度数据
- 示例: 前 10 个产品及其收入、销售单位、类别

**KPI 可视化**
- 用于: 总收入、总订单数、平均订单价值
- 显示: 带比较的单个关键指标
- 示例: 本月收入与上月对比

### 2. 计算字段

直接在 QuickSight 中创建自定义指标:

**利润率**
```
(revenue - cost) / revenue * 100
```

**每客户收入**
```
sum(total_revenue) / distinctCount(customer_id)
```

**订单转化率**
```
count(completed_orders) / count(total_orders) * 100
```

**同比增长**
```
(current_year_revenue - previous_year_revenue) / previous_year_revenue * 100
```

### 3. 筛选器

**日期筛选器**
- 最近 7 天、最近 30 天、上季度
- 自定义日期范围
- 相对日期筛选器(例如,"本月")

**维度筛选器**
- 地区: 筛选到特定地区
- 客户细分: 专注于企业、中小企业等
- 产品类别: 分析特定类别
- 订单状态: 仅显示已完成订单

**前/后 N 筛选器**
- 按收入排名前 10 的产品
- 按销售额排名后 5 的地区
- 前 20% 的客户(按收入)

### 4. 参数

使用用户控件创建交互式仪表板:

**收入阈值**
- 让用户筛选高于特定收入的产品
- 控件: 滑块或输入框

**日期范围**
- 允许用户选择自定义日期范围
- 控件: 日期选择器

**地区选择**
- 地区的多选下拉列表
- 控件: 下拉列表

### 5. 下钻

创建分层导航:

**地理下钻**
- 地区 → 国家 → 城市

**产品下钻**
- 类别 → 子类别 → 产品

**时间下钻**
- 年 → 季度 → 月 → 日

### 6. 预测

QuickSight ML 驱动的预测:

**收入预测**
- 预测未来 30 天的收入
- 基于历史每日销售趋势
- 显示置信区间

**需求预测**
- 预测产品需求
- 帮助库存规划

### 7. 异常检测

自动检测异常模式:

**收入异常**
- 检测意外的峰值或下降
- 获取重大变化的警报

**订单量异常**
- 识别异常订单模式
- 调查潜在问题

### 8. 条件格式

突出显示重要数据:

**收入目标**
- 绿色: 高于目标
- 黄色: 接近目标
- 红色: 低于目标

**利润率**
- 从红色(低)到绿色(高)的颜色刻度

**订单状态**
- 已完成、待处理、已取消的不同颜色

## 示例仪表板布局

### 执行仪表板

**第 1 行: KPI**
- 总收入(当月)
- 总订单数(当月)
- 平均订单价值
- 客户数量

**第 2 行: 趋势**
- 每日销售趋势(折线图)
- 按地区收入(条形图)

**第 3 行: 分析**
- 客户细分绩效(表格)
- 前 10 个产品(条形图)

### 产品绩效仪表板

**第 1 行: 概览**
- 已售产品总数
- 总收入
- 平均利润率

**第 2 行: 类别分析**
- 类别绩效(带利润的条形图)
- 类别组合(饼图)

**第 3 行: 详情**
- 热门产品表格(包含收入、单位、利润)
- 产品趋势(折线图)

### 客户分析仪表板

**第 1 行: 指标**
- 总客户数
- 每客户收入
- 每客户平均订单数

**第 2 行: 细分**
- 客户细分分布(饼图)
- 细分绩效(表格)

**第 3 行: 行为**
- 订单频率分布
- 客户生命周期价值

## 最佳实践

1. **从简单开始**: 从基本图表开始,根据需要增加复杂性
2. **使用筛选器**: 使用筛选器和参数使仪表板具有交互性
3. **颜色一致**: 在所有可视化中对相同维度使用相同颜色
4. **添加上下文**: 包括比较期间(与上月、与去年对比)
5. **优化性能**: 使用 SPICE 以获得更快的查询
6. **移动友好**: 设计适用于移动设备的仪表板
7. **分享见解**: 发布仪表板并设置电子邮件报告

## 后续步骤

1. 将数据集从 Redshift 导入 QuickSight
2. 为利润率和增长率创建计算字段
3. 使用 3-4 个关键可视化构建您的第一个仪表板
4. 添加日期范围和地区的筛选器
5. 为利益相关者设置定期电子邮件报告
6. 启用 ML 见解以进行预测和异常检测
